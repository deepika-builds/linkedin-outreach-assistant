# LinkedIn Outreach Assistant

A Chrome extension that generates personalized LinkedIn outreach messages using Claude AI. It scrapes the profile you're viewing and produces ready-to-send drafts that reference specific details — not generic templates.

## What it does

- Injects a side panel on any LinkedIn profile page (`/in/*`)
- Scrapes name, headline, location, about, experience, education, and skills
- Generates 1–3 tailored outreach drafts via the Claude API
- Supports connection requests, InMail, and follow-up message types
- Quick actions to make a draft shorter, longer, more formal, or more personal
- PDF resume upload and URL portfolio fetching (GitHub profiles, personal sites)

## Setup

### 1. Install dependencies and build

```bash
npm install
node build.js
```

This bundles the source files into `dist/`. You need to rebuild after any source changes.

For development with auto-rebuild on save:

```bash
node build.js --watch
```

### 2. Load the extension in Chrome

1. Go to `chrome://extensions`
2. Enable **Developer mode** (top right)
3. Click **Load unpacked**
4. Select the root of this repo (the folder containing `manifest.json`)

### 3. Add your Claude API key

1. Click the extension icon → **Open Settings**
2. Paste your Claude API key (`sk-ant-...`) — get one at [console.anthropic.com](https://console.anthropic.com)
3. Click **Validate**, then **Save Settings**

## Configuration (Settings page)

| Field | Description |
|---|---|
| **Resume / Work History** | Your background. Paste text or upload a PDF — Claude extracts the key details. |
| **Current Focus / Pitch** | What you're looking for right now. Anchors every message to your goal. |
| **Voice / Writing Samples** | 2–5 past messages that felt authentic. Claude matches this voice. |
| **Portfolio URLs** | GitHub profile or personal site — fetched and summarized automatically. |
| **Company Context** | Optional. Describe your company if you're reaching out on its behalf. |
| **Message Defaults** | Tone, length, and draft count applied to every generation. |

## How the scraper works

The scraper (`scraper.js`) uses a priority-ordered extractor approach for each field. It tries multiple strategies in sequence and returns the first non-empty result:

1. `aria-label` on `<section>` elements — the current LinkedIn a11y standard
2. Element `id` or `data-*` attributes
3. `h2` heading text — walks up the DOM to find the owning container
4. `div[aria-label]` fallback

This makes it resilient to LinkedIn's frequent class name changes. If a scrape misses a field, open DevTools on the LinkedIn tab — the scraper logs every field and any missing sections via `console.group('[LIA] Scraped profile')`.

## Rebuilding after changes

Any change to `src/` requires a rebuild before it takes effect in the extension:

```bash
node build.js
```

Then reload the extension at `chrome://extensions` (click the refresh icon on the extension card).

## Permissions used

| Permission | Why |
|---|---|
| `activeTab` | Access the current LinkedIn tab |
| `storage` | Save settings and API key |
| `scripting` | Inject the overlay panel |
| `host_permissions: linkedin.com` | Run the content script on profile pages |
| `host_permissions: api.anthropic.com` | Call the Claude API from the service worker |
